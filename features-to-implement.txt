Current implementation:
- Webhook receives Gmail emailAddress
- Iterates through all Gmail tenants
- Calls tenant.gmail.api.users.getProfile() repeatedly
- Builds an in-memory email → tenant cache

Problems:
- O(n) Gmail API calls on cold start
- Cache is lost on server restart
- Doesn't scale to many tenants
- Unnecessary Gmail API traffic

Future implementation:
1. During Gmail OAuth callback:
   - Call users.getProfile({ userId: "me" })
   - Extract emailAddress
   - Persist:
       tenantId
       gmailEmail
   - Store in dedicated table

2. During webhook processing:
   - Decode Pub/Sub payload
   - Extract emailAddress
   - Query DB:
       SELECT tenantId
       FROM gmail_connections
       WHERE gmailEmail = ?
   - Pass resolved tenantId into processWebhook()

Benefits:
- O(1) lookup
- Single DB query
- No Gmail API calls
- Survives restarts
- Scales to thousands of users
